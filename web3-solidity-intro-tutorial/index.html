回顧智能合約 - 簡介

智能合約 (Smart Contract） 本質上是一組程式碼的集合，包含兩個東西：

  1. 函數 : 用來執行動作，例如存資料、取資料
  2. 資料 : 儲存在區塊鏈上的變數與資訊

當我們把智能合約部署（Deploy）到以太坊區塊鏈上，它就會被分配一個唯一的區塊鏈位址。

任何人都可以透過這個位址來與合約互動。

解釋一些左邊部署的參數:

  • Account（帳號） → 系統會提供一些測試帳號，每個帳號都有測試用的 Ether，可以隨意操作
  • Gas Limit → 設定交易的最大 Gas 消耗上限，保持預設值即可
  • Value → 預設為 0，因為我們目前不需要傳送 Ether
  • Contract → 選擇我們剛剛編譯好的合約，應該會看到 simpleTest

解釋 deploy 後，下方的 console 顯示的訊息

  • From → 部署的帳戶地址（例如 0x5B3…）
  • Transaction Cost（交易成本） → 使用了一些測試 Ether 作為 Gas 費
  • Gas Used（實際消耗的 Gas） → 小於設定的 Gas Limit
  • Value → 0，因為我們沒有傳送 Ether
  • Transaction Hash（交易雜湊值） → 每筆交易的唯一編號，可以複製下來

最後對 deploy 後的合約進行測試

如下圖可以看到左邊是我們設定的函數，每次執行就可以看到下面的 console 會有訊息出來，就是我們的交易資訊

  • FROM: 交易來源帳號 (就是平台給的測試帳號)
  • TO: 我們合約地址
  • GAS: 交易手續費
